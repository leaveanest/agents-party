Architecture

Slack境界からprovider adapterまでを分ける。

Agents partyはTypeScriptのlayered modular monolithです。Slack SDK、AI SDK、PostgreSQL SDKの型を 必要な境界に閉じ込め、ドメイン履歴はリポジトリ独自モデルで扱います。

Slackevents, interactions, OAuth
Agentsrouting, tools, specialist runners
Domainmessage history, route models
ProvidersAI SDK adapters, native escape hatches
PostgreSQLinstallations, credentials, migrations

Slack IDはworkspace scoped

channel、user、thread、message timestampは `team_id` と組み合わせて扱い、グローバルIDのように保存しません。

AI SDKは呼び出し境界に閉じる

AI SDKの `ModelMessage[]` をドメイン履歴として保存せず、provider invocation時に変換します。

Provider routingは自前で持つ

model registry、capability matrix、native provider escape hatchをrepository-owned moduleとして管理します。

Redis workerでSlack ackを分離

必要に応じてSlack ingressとprovider executionをqueueで分け、重い処理をworkerに渡します。

データ境界はdocsへ

保存対象、provider送信データ、秘密情報の扱いはREADME、data-processing docs、 SECURITY.mdに集約します。

PlantUML diagrams live in the repository docs.
