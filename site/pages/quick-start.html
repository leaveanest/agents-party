Quick Start

Slackなしでも、まずはヘルスチェックまで起動する。

本番運用にはSlack App、PostgreSQL、プロバイダーキーが必要です。最初はHTTPサーバーの起動から確認します。

依存関係を同期

このリポジトリではJavaScript/TypeScriptの通常操作に `vp` を使います。

vp install

サーバーを起動

Slack設定なしでも `GET /healthz` を確認できます。

vp run dev
curl http://localhost:8000/healthz

PostgreSQLを追加

Slack installation、OAuth state、workspace routing、credential storageにはPostgreSQLが必要です。 

docker compose up -d postgres
export DATABASE_URL=postgresql://agents_party:agents_party@localhost:5432/agents_party
vp run migrate

Slack App Manifestを設定

`slack-app-manifest.yaml` のURLを公開HTTPSトンネルに置き換え、Slackワークスペースにインストールします。

詳細はGitHubのREADMEを参照してください。
