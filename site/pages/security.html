Security & Data

何を保存し、何を保存しないかを分けて説明する。

Slack内のAI体験はデータ境界が曖昧に見えやすい領域です。Agents partyでは、運用設定、 OAuth state、workspace credentials、provider送信データを分けて扱います。

Stored

運用に必要な設定

Slack installation、workspace route、OAuth state、暗号化されたworkspace credentialsはPostgreSQLに保存します。

Not a transcript store

会話ログ保管庫ではない

AI SDK message配列をドメイン履歴として保存しません。provider呼び出し境界で必要な形式へ変換します。

Operator duty

外部サービス条件を確認

Slack、LLM provider、OAuth provider、object storageの利用規約とデータ処理条件は導入者が確認します。

公開サイト、Issue、PR、ログ、スクリーンショットにはSlack token、OAuth secret、provider API key、 database URL、Terraform state、顧客データを含めないでください。
Review repository security docs before operating with workspace data.
