Slack-native agent routing OSS

Slackの会話に、AIエージェントが自然に参加する。

Agents partyは、Party on Slackの思想を受け継ぐOSS基盤です。仲間、AI、自分が同じSlackの作業導線で会話し、 要約、検索、翻訳、ファイル理解、専門ツール呼び出しを進められるようにします。

設計思想

AIを別アプリに閉じ込めず、チームがすでに使っているSlackの会話空間に参加させる。 Agents partyは、その思想をOSSとして運用できる形に分解します。

Conversation

仲間・AI・自分で進める

DMだけではなく、チャンネルやスレッドの文脈でAIが会話に参加する前提で設計します。

Routing

モデルを固定しない

OpenAI、Anthropic、Google、Vertex AIなど複数プロバイダーを、用途やワークスペース設定で切り替えます。

Operator control

運用者が制御できる

Slack OAuth、workspace credentials、PostgreSQL、Terraformを明示し、導入者が責任範囲を把握できます。

公開初期の入口

OSS利用者が最初に必要とする導入と構造に短い導線を置きます。

Quick Start

Docker Composeで起動し、必要に応じてVite+ CLIの vp で検証します。
