Quick Start

Start with a local health check before connecting Slack.

Production use requires a Slack app, PostgreSQL, and provider credentials. The first local path starts with the HTTP server.

Install dependencies

This repository uses `vp` for JavaScript and TypeScript workflows.

vp install

Run the server

The server exposes `GET /healthz` even before Slack credentials are configured. 

vp run dev
curl http://localhost:8000/healthz

Add PostgreSQL

Slack installations, OAuth state, workspace routing, and encrypted credentials require PostgreSQL. 

docker compose up -d postgres
export DATABASE_URL=postgresql://agents_party:agents_party@localhost:5432/agents_party
vp run migrate

Configure the Slack App Manifest

Update `slack-app-manifest.yaml` with your public HTTPS tunnel and install it into a Slack workspace.

See the GitHub README for the complete operator path.
