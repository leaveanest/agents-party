Quick Start

Start with Docker Compose and verify web plus worker together.

compose.yaml is the fastest local path for Agents party. It can start PostgreSQL, Redis, migrations, web, and worker together even before Slack credentials are configured.

Required

Local prerequisites

Use Node.js 22.x, vp, Docker Compose, and a Slack workspace where you can create or install an app.

Do not switch

Package workflow

Use vp when working outside containers for dependency sync, validation, and builds. Do not switch normal workflows to direct npm, pnpm, or yarn commands.

Secrets

Keep secrets out

Do not put Slack tokens, OAuth secrets, provider API keys, database URLs, or copied Slack transcripts in public issues, PRs, logs, or screenshots.

Verify with Compose

Start with the no-Slack smoke test, then add the bootstrap workspace route and real Slack credentials.

Start web and worker

Starting web and worker also starts PostgreSQL, Redis, and migrations through service dependencies. Health checks work without Slack credentials. 

docker compose up --build web worker

Check the health endpoint

In another shell, confirm that the HTTP server is responding. At this point the app build, migrations, and web process have all run. 

curl http://localhost:8000/healthz

Seed the first workspace route

App mentions need an enabled agent and model route. For real Slack testing, set AGENTS_PARTY_BOOTSTRAP_TEAM_ID to the actual Slack workspace team id. 

AGENTS_PARTY_BOOTSTRAP_TEAM_ID=T123456789 \
docker compose --profile seed run --rm seed

Prepare .env for real Slack traffic

Replace agents-party.example.com in the Slack App Manifest with a public HTTPS tunnel that forwards to http://localhost:8000, then import it into Slack. Compose reads local settings from .env. 

cp .env.example .env
# Fill SLACK_SIGNING_SECRET, SLACK_CLIENT_ID, SLACK_CLIENT_SECRET,
# SLACK_STATE_SECRET, AGENT_MODEL, and provider credentials as needed.
docker compose up --build web worker

Watch web and worker logs

In queue mode, the web process receives Slack requests while the worker performs agent execution and posts replies. 

docker compose logs -f web
docker compose logs -f worker

Stop or reset the stack

Only remove volumes when you intentionally want to reset local PostgreSQL and Redis data. 

docker compose down
docker compose down -v  # reset local PostgreSQL and Redis volumes

Running pieces outside containers

When you want to run only the web process on the host, use vp install, keep PostgreSQL and Redis in Compose, then run vp run dev or vp run worker locally.

Validation before a PR

Run the normal repository validation set before opening a pull request.

vp check
vp run typecheck
vp test
vp pack
