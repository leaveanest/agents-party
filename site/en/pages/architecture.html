Architecture

Keep Slack ingress, agent routing, and provider adapters separate.

agents-party is a layered TypeScript modular monolith. Slack SDK, AI SDK, and PostgreSQL SDK types stay at their boundaries while domain message history remains repository-owned.

Slackevents, interactions, OAuth
Agentsrouting, tools, specialist runners
Domainmessage history, route models
ProvidersAI SDK adapters, native escape hatches
PostgreSQLinstallations, credentials, migrations

Slack IDs are workspace scoped

Channel, user, thread, and message timestamp identifiers are stored and compared with `team_id`.

AI SDK stays at invocation boundaries

The app does not store AI SDK `ModelMessage[]` as the domain history format.

Provider routing is repository-owned

Model registry, capability matrix, and native provider escape hatches stay in local TypeScript modules.

Redis workers separate Slack acknowledgements

Queue mode lets Slack ingress acknowledge quickly while provider execution runs in workers.
