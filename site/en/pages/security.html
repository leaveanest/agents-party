Security & Data

Separate what is stored from what is only sent to providers.

Slack-native AI tools can make data boundaries feel blurry. agents-party separates runtime settings, OAuth state, workspace credentials, and data sent to external providers.

Stored

Operational configuration

Slack installations, workspace routes, OAuth state, and encrypted workspace credentials are stored in PostgreSQL.

Not a transcript store

No AI SDK history persistence

The app does not store AI SDK message arrays as the domain history format.

Operator duty

Review external service terms

Operators must review Slack, LLM provider, OAuth provider, and object storage data terms.

Do not put Slack tokens, OAuth secrets, provider API keys, database URLs, Terraform state, customer data, or copied Slack transcripts in public issues, pull requests, logs, or screenshots.
Review repository security docs before operating with workspace data.
